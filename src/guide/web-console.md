# 网页控制台

Pallas-Bot 自带的运维界面：看在线状态、管协议端、改插件与业务配置、装扩展、查日志。**不用单独起服务**——`uv run pallas`（或 Docker）启动后即可打开。

日常改配置优先走控制台；哪些项仍要改 `pallas.toml`，见 [配置从哪改](config.md)。

::: tip
手机或窄窗口也能用。侧栏可收起；表格类页面可横向滑动。
:::

## 打开与登录

| 项 | 值 |
| --- | --- |
| 地址 | `http://<主机>:8088/pallas/` |
| 端口 | `pallas.toml` → `[bootstrap] port`，默认 **8088** |
| 首次密码 | 启动日志打印；口令文件在 `data/pallas_console/` |

1. 浏览器打开上表地址（本机可用 `127.0.0.1`）
2. 输入启动日志中的控制台密码
3. 能看到首页运行概览，侧栏有「插件」「运行日志」= 登录成功

::: warning
不要长期开开发模式（会跳过登录）。公网务必 HTTPS + 强密码，勿把控制台裸暴露到公网。
:::

::: details 忘了密码 / 换密码
- 忘了：见 [FAQ · 控制台密码](/deploy/faq)（按页内说明重置）
- 换密码：登录后在控制台账号 / 安全相关入口修改（以当前版本界面为准）
:::

## 装完先做这几步

Bot 已能打开控制台后，按这个顺序把「能用」补齐：

| 顺序 | 去做 | 侧栏 / 文档 |
| --- | --- | --- |
| 1 | 协议端上线（扫码或自管 NapCat） | **协议端实例** · [连接 QQ](connect-qq.md) |
| 2 | 给这只牛配号主 | **实例与连接** · [号主](bot-owner.md) |
| 3 | 需要时装官方玩法扩展 | **插件商店** · [安装插件](install-plugins.md) |
| 4 | 改 AI、语料、权限等业务项 | **通用配置** / **命令权限** · [配置从哪改](config.md) |

群里发 **牛牛帮助** 能出图，说明账号与权限链路大致正常。

## 侧栏导览

| 你想… | 去哪 |
| --- | --- |
| 看是否在线、消息量 | **仪表盘** |
| 配号主、看框架里有哪些牛 | **实例与连接** |
| 管 NapCat / 协议端 | **协议端实例**（或协议端管理） |
| 改某个插件配置 | **插件** → 点插件名 |
| 全实例关掉某插件 | **插件** → **全实例禁用** |
| 装 / 卸决斗、MAA 等 | **插件商店** |
| 改命令谁能用 | **命令权限** |
| 改 AI / 语料 / 外部服务 | **通用配置** |
| 看报错与运行日志 | **运行日志** / **日志报错** |
| 备份、清理库 | **数据库** |

完整日常操作清单也可对照 [日常管理](usage-admin.md)。

## 保存与重启

| 操作 | 通常要不要重启 |
| --- | --- |
| 插件页点保存、通用配置、命令权限 | **不用**（已接热重载的多数立刻生效） |
| 插件商店装 / 卸 / 升级官方扩展 | **必须重启 Bot** |
| 改 `pallas.toml` 端口、数据库、超管 | **必须重启**，且不在控制台里改 |

网页点保存 → 写入 `data/pallas_config/webui.json`。同名项会覆盖 `pallas.toml`；排障时两处都看。细节见 [配置从哪改](config.md)。

::: details 开发向：合并顺序
`pallas.toml` → `.env` → `webui.json`（后者优先）。合同见 [配置存储](/developer/architecture/config-storage)。
:::
