# WebUI 运维

## 三层结构

| 层 | 路径 |
| --- | --- |
| 前端源码 | `Pallas-Bot-WebUI` |
| 运行产物 | `data/pb_webui/public/` |
| 后端 | `pb_webui` · `/pallas/api` |

现象相似时，按层排查，处理方式不同。

## 按现象检查

### 页面打不开

- Bot 是否已启动
- `pb_webui` 是否挂载
- 路径是否为 `/pallas/`

### 页面能打开，内容像旧版

- `data/pb_webui/public/` 是否为新产物
- 浏览器是否强制刷新
- 是否只改了源码仓、未同步运行产物

### 页面能打开，保存失败

- `/pallas/api` 是否可达
- 写接口是否返回鉴权错误或 500
- 对应插件是否支持当前配置保存

## 依赖插件的页面

部分控制台页不只依赖 `pb_webui`：

| 页面类型 | 额外依赖 |
| --- | --- |
| 协议端 | 协议端插件 |
| 插件页 | 插件 metadata 与能力聚合 |
| AI | AI 运行态接口与聚合逻辑 |

页面空白时，除前端资源外，也查后端插件是否加载。

## 何时查前端仓

优先查 `Pallas-Bot-WebUI`：

- 布局错乱
- 组件未渲染
- 按钮位置或样式异常
- 窄屏适配问题

## 何时查主仓

- API 返回异常
- 保存配置失败
- 运行态数据不对
- 插件列表、权限、治理信息不完整

## 检查顺序

1. 页面路径与登录
2. 运行产物是否同步
3. `/pallas/api/health` 与基础接口
4. 对应后端插件是否加载
5. 再区分前端或后端
